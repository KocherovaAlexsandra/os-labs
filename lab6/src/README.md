# Алгоритм сортировки пузырьком (Bubble Sort)

**Суть алгоритма:**
Алгоритм многократно проходит по структуре данных, попарно сравнивает соседние элементы и меняет их местами, если они стоят в неправильном порядке.
Крупные элементы "всплывают" к концу массива, маленькие — "проваливаются" к началу.

### Характеристики:

| Сценарий                          | Сложность |
| --------------------------------- | --------- |
| Лучший случай (уже отсортировано) | `O(N)`    |
| Худший случай                     | `O(N²)`   |
| Средний случай                    | `O(N²)`   |

# Описание работы обновлённой программы

Программа сравнивает скорость выполнения пузырьковой сортировки на трёх структурах данных.
В обновлённой версии был добавлен механизм проведения **50 повторных экспериментов для каждого размера данных**, что позволяет получить статистически корректные результаты.

## 1. Подготовка входных данных

Для каждого размера (`100`, `1000`, `5000`, `10000`) программа:

1. 50 раз генерирует массив случайных чисел методом `generateArray()`;
2. копирует один и тот же набор данных в три структуры:

   * `ArrayList<Integer>`
   * `LinkedList<Integer>`
   * `ArrayDeque<Integer>`
3. сортирует данные для каждой структуры;
4. сохраняет время выполнения.

Так обеспечивается честное сравнение результатов.

---

## 2. Проведение 50 экспериментов

Для каждого размера выполняются:

* 50 сортировок `ArrayList`,
* 50 сортировок `LinkedList`,
* 50 сортировок `ArrayDeque`.

Результаты времени суммируются и затем усредняются:

```
среднее = сумма_всех_замеров / 50
```

Это исключает влияние случайных выбросов (например, работы GC или фоновый процесс ОС).

---

## 3. Алгоритмы сортировки

### **ArrayList**

Сортировка выполняется напрямую, через `get()` и `set()`.
ArrayList подходит для пузырька лучше всего, так как предоставляет быстрый доступ по индексу.

---

### **LinkedList**

Прямой доступ по индексу отсутствует — он затратный (`O(n)`), поэтому:

1. список копируется в массив;
2. массив сортируется пузырьком (`O(N²)`);
3. список очищается и заполняется отсортированными элементами.

Это обеспечивает логически корректную сортировку при минимальных затратах.

---

### **ArrayDeque**

Аналогично `LinkedList`:

1. элементы копируются в массив;
2. массив сортируется пузырьком;
3. `ArrayDeque` заполняется обратно.

---

## 4. Измерение времени

Для времени выполнения используется:

```
System.currentTimeMillis()
```

Замеряется только:

* сортировка для `ArrayList`;
* копирование → сортировка → восстановление для `LinkedList` и `ArrayDeque`.

Для каждой структуры данных вычисляется **среднее из 50 экспериментов**.

# Аналитическая оценка сложности

Пузырьковая сортировка имеет квадратичную сложность:

* **Средняя сложность:** `O(N²)`
* **Ниже средней скорости для больших массивов**
* Не рекомендуется для реальных задач сортировки

**ArrayList** показывает лучшую производительность, т.к. операции доступа `get()` и `set()` выполняются за `O(1)`.

**LinkedList** и **ArrayDeque** требуют дополнительных затрат на преобразование в массив, однако итоговое время всё равно определяется квадратичной сложностью алгоритма сортировки.
